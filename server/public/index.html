Kiro credentials

        
Each credential = 1 Kiro account (free tier = 50 credits/month + 500 bonus). Stored in kiro_credentials.json (gitignored). Tokens auto-refresh ~1 min before expiry. Cooldown after rate-limit, auto-recovers.


        

            
Add credential

            
Run kiro-cli login --license free --use-device-flow on a machine that ran Kiro IDE; then extract from ~/.local/share/kiro-cli/data.sqlite3 (Linux) or ~/Library/Application Support/kiro-cli/data.sqlite3 (macOS):

            sqlite3 data.sqlite3 "SELECT value FROM auth_kv WHERE key='kirocli:odic:token';"

Temp Mail

            

                
            

        

        

            Cloudflare Email Routing forwards *@yourdomain.com to a Gmail account; sambungin polls
            that Gmail over IMAP every 10s and routes mail to the matching temp address.
            All state (inboxes, domains, addresses, message cache) persists in server/tempmail.json
            so a restart never wipes your config.
        


        
        

            
Setup guide


            
    
                
  1. 
                    Cloudflare Email Routing (per domain):
                    
      
                        
    1. Open the domain on Cloudflare → Email → Email Routing → Get started.
      2. 
                        
    2. Cloudflare auto-adds the required MX + SPF records — accept them.
      3. 
                        
    3. Add a destination address (the Gmail you want everything to land in) and click the verification link Cloudflare emails to it.
      4. 
                        
    4. Open Routing rules → enable Catch-all address → action Send to an email → pick that destination Gmail. Save.
      5. 
                    
    
                
    2. 
                
  2. 
                    Gmail App Password (per Gmail account, only once):
                    
      
                        
    1. Enable 2-Step Verification on the Gmail account.
      2. 
                        
    2. Go to myaccount.google.com/apppasswords, create an app password named e.g. sambungin.
      3. 
                        
    3. Copy the 16-character string (no spaces). Paste it as App password below.
      4. 
                    
    
                
    3. 
                
  3. 
                    Add the Gmail inbox below, then add the domain and link it to that inbox.
                    Once both are saved, generate temp addresses with the form at the bottom and start polling.
                
    4.

Content Filter

        

        

            Pattern-based string replacement applied to requests before forwarding to upstream.
            Use this to strip fingerprints (billing headers, identity strings, entrypoint markers) that cause upstream providers to refuse requests.
            Action remove = replace with empty string. You can also specify a custom replacement.
